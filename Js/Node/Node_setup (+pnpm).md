1. Если стоит homebrew или другой мененджер пакетов в системе - убрать конфликт (удалить node homebrew или просто решить конфликт что бы он не спольщзовался.)

```bash
brew uninstall node
brew cleanup
# проверим:
which node   # должно ничего не найти (пока)
node -v      # "command not found" — это нормально на этом этапе
```

2. Установить nvm (официальный скрипт)

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

Перезагрузи оболочку, чтобы подхватились настройки:
```bash
exec $SHELL -l
```

Проверка:	
```bash
nvm -v
```

3. Поставить последний LTS (рекомендую для разработки)

```bash
nvm install --lts
nvm alias default lts/*   #сделать так, чтобы при открытии нового терминала всегда включалась последняя LTS-версия.
```

Проверка:
```bash
node -v   # должно показать v22.20.0 (на сегодня)
npm -v
which node  # путь должен быть вида: /Users/твоё_имя/.nvm/versions/node/v22.20.0/bin/node
```

4. Включить pnpm через Corepack (современно и чисто)
В Node 22+ есть Corepack — менеджер, который активирует pnpm/yarn без глобальной установки:

```bash
corepack enable
corepack prepare pnpm@latest --activate
pnpm -v		#Теперь во всех проектах можешь использовать pnpm без npm install -g pnpm.
```


5. Автоподхват версии в проекте (.nvmrc)

Чтобы при входе в папку проекта автоматически включалась нужная версия:
В корне проекта создай файл:

```bash
echo "lts/*" > .nvmrc
# или зафиксируй конкретную мажорную/патч-версию, например:
# echo "22.20.0" > .nvmrc
```

При заходе в папку проекта просто выполняй:

```bash
nvm use  # (Если работаешь с несколькими проектами с разными версиями Node)
```

(Если хочешь автопереключение без команды — добавлю zsh-хук или подскажy лёгкий плагин.)
